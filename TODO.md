## 四、实现原则

### 代码实现原则

- [x] 不增加和当前课题无关的抽象层
- [x] 不把单区域原型提前过度设计成通用平台
- [x] 新逻辑优先写在已有模块内，保持目录结构稳定
- [x] 日志和导出字段命名统一
- [x] 所有 fallback 都必须可见，不允许静默降级

### 数据使用原则

- [x] 现有数据中已经存在的关键字段，至少进入“摘要 / 导出 / 结果解释”之一
- [x] 没有数据支撑的能力，不假装实现
- [x] 需要未来采集的数据，用文档标记为后续项，不混入当前完成标准

### 实验表达原则

- [x] 结果必须能回答课题问题，而不是只输出模型分数
- [x] 每个 region 的结论都要能回溯到 `IFI / MDI / IAI`
- [x] 结果必须能说明当前使用的数据范围和标签完备度
