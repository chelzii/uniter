# UNITER 单区域 Base 技术文档

## 一、系统定位

当前版本面向单区域“失落空间”研究场景，采用区域级多模态建模方式，对空间、身份、意义三个维度进行联合分析，并输出区域级失落空间判定结果。系统以街景图、卫星图、当前感知文本、历史与规划文本、身份文本及区域级标签为输入，在共享潜空间中完成图像与文本表征对齐，并在此基础上构建 `IFI`、`IAI`、`MDI` 三个核心指标。仓库级使用说明见 [README.md](/home/chelizi/project/uniter/README.md)，本文只展开技术结构与当前实现边界。

系统默认采用规则判定作为最终输出路径。`lost_space` 分类头在代码中保留，但默认不参与最终结论生成。当前实现适用于单区域 bootstrap 研究与案例分析，不作为多区域统计推断系统使用。

## 二、总体架构

系统采用区域级 Encode-Align-Decode 结构。图像分支对街景图和卫星图进行空间表征提取，文本分支对当前文本、历史文本和身份文本进行语义编码，之后将图像与文本投影到共享表示空间。共享空间中的图像嵌入、文本嵌入和分支级中间特征进一步进入空间、身份、意义三个指标模块。最终由规则判定模块根据三指标及阈值配置输出区域级失落空间等级。

主模型位于 [multimodal.py](/home/chelizi/project/uniter/src/uniter/models/multimodal.py)。配置系统位于 [config.py](/home/chelizi/project/uniter/src/uniter/config.py) 和 [base.yaml](/home/chelizi/project/uniter/configs/base.yaml)。命令行入口位于 [cli.py](/home/chelizi/project/uniter/src/uniter/cli.py)。

从实现层面看，系统由数据层、编码层、指标层、判定层和输出层构成。数据层负责 manifest、图像元数据、文本与标签的统一组织；编码层负责图像和文本向量化；指标层负责构造 `IFI / IAI / MDI`；判定层负责等级映射与阈值应用；输出层负责评估、结果表、可视化与上下文解释。

## 三、数据模型与输入结构

系统以“区域”为样本单位。训练入口默认使用 [regions.jsonl](/home/chelizi/project/uniter/data/regions.jsonl)。每个 `region_id` 下可以聚合多张图像、多条当前文本、多条历史文本以及若干身份文本，并在区域级别汇总输出指标和结论。

当前 manifest 结构由 [manifest.py](/home/chelizi/project/uniter/src/uniter/data/manifest.py) 定义，数据集加载逻辑位于 [dataset.py](/home/chelizi/project/uniter/src/uniter/data/dataset.py)，批处理组装位于 [collate.py](/home/chelizi/project/uniter/src/uniter/data/collate.py)。

图像输入由 `image_paths` 提供，系统支持同一区域内同时使用街景图和卫星图。文本输入分为三组：`current_texts` 对应现实感知材料，`historical_texts` 对应历史与规划叙事材料，`identity_texts` 对应场所身份表达材料。标签输入包括逐文本当前情感标签、逐文本历史情感标签及区域级 `lost_space_label`。标签导入逻辑位于 [annotations.py](/home/chelizi/project/uniter/src/uniter/data/annotations.py)。

图像元数据通过图像索引文件接入，当前在 batch 中保留 `point_id`、`view_direction`、`lon`、`lat` 和 `image_type` 等字段。空间分支据此区分街景与卫星视角，并在 `IFI` 中显式使用点位几何和视角信息。

当前单区域数据目录由三层组成。第一层是原始或整理中的工作区 [data_workspace](/home/chelizi/project/uniter/datasets/data_workspace)，第二层是已经完成清洗、筛选和结构化后的训练工作区 [data_workspace_cleaned](/home/chelizi/project/uniter/datasets/data_workspace_cleaned)，第三层是独立保存人工标注结果的 [annotations](/home/chelizi/project/uniter/datasets/annotations)。训练与推理默认读取的是清洗后工作区对应的路径，而不是原始工作区。

清洗后工作区内部目前包含 `images`、`current_texts`、`historical_texts`、`identity_texts` 和 `metadata` 五类目录。图像目录 [images](/home/chelizi/project/uniter/datasets/data_workspace_cleaned/images) 中同时保存街景图和卫星图，区域级索引文件是 [image_index.csv](/home/chelizi/project/uniter/datasets/data_workspace_cleaned/images/image_index.csv)。当前文本目录 [current_texts](/home/chelizi/project/uniter/datasets/data_workspace_cleaned/current_texts) 中保留了清洗后文本及其索引文件 [current_text_index_cleaned.csv](/home/chelizi/project/uniter/datasets/data_workspace_cleaned/current_texts/current_text_index_cleaned.csv)，其中已经包含 `source_platform`、`normalized_date`、互动量和清洗保留状态等字段。历史与规划文本、身份文本和元数据也分别存放在对应目录中，并由 manifest 统一引用。

人工标注目录 [annotations](/home/chelizi/project/uniter/datasets/annotations) 按区域独立组织。当前单区域 `kaitong_west_lane` 下已经包含当前文本情感标注 [current_sentiment_labels.csv](/home/chelizi/project/uniter/datasets/annotations/kaitong_west_lane/current_sentiment/current_sentiment_labels.csv)、历史文本情感标注 [historical_sentiment_labels.csv](/home/chelizi/project/uniter/datasets/annotations/kaitong_west_lane/historical_sentiment/historical_sentiment_labels.csv) 以及区域级失落空间标签 [region_lost_space_label.json](/home/chelizi/project/uniter/datasets/annotations/kaitong_west_lane/lost_space/region_lost_space_label.json)。这些标注不会直接替代 manifest，而是通过导入逻辑写回训练清单，从而保持数据源、标注源和训练入口之间的分层结构。

`data` 目录中的 [regions.jsonl](/home/chelizi/project/uniter/data/regions.jsonl)、[regions.single_region.full.jsonl](/home/chelizi/project/uniter/data/regions.single_region.full.jsonl) 和 [regions.annotated.preview.jsonl](/home/chelizi/project/uniter/data/regions.annotated.preview.jsonl) 承担的是训练入口、全量单区域清单和预览性输出等不同职责。也就是说，`datasets` 保存的是原始工作区、清洗工作区和人工标注，`data` 保存的是最终进入训练流程的区域级 manifest 及其中间产物。

在实现上，数据进入训练流程时会经过两层组织。第一层是记录级组织，即把文本、图像和元数据读成统一的区域记录对象；第二层是 batch 级重排，即在 `collate` 阶段把多图像、多文本、多视角和多标签重新展开并生成索引张量。系统正是在这一层把“区域级样本”转换成模型可以处理的张量表达，同时保留记录级回溯能力，以便后续评估和导出可以重新对应回具体区域、具体文本和具体点位。

## 四、编码层实现

图像编码器位于 [models/spatial.py](/home/chelizi/project/uniter/src/uniter/models/spatial.py)。当前默认骨干为 `SegFormer`，用于提取区域级图像语义特征和分割 logits。系统并不直接使用分割结果做最终判定，而是将其进一步转换为空间类别占比、街景/卫星形态代理以及空间代理监督信号。

文本编码器位于 [models/text.py](/home/chelizi/project/uniter/src/uniter/models/text.py)。当前默认骨干为 `MacBERT`。当前文本、历史文本和身份文本共用同一文本编码器，但在区域级聚合后进入不同下游分支。共享编码保证了三个文本来源在同一语义空间中可比较，同时降低了分支间表示漂移。

图像与文本编码后的区域级特征通过投影头映射到共享潜空间。投影头实现位于 [projection.py](/home/chelizi/project/uniter/src/uniter/models/projection.py)。共享潜空间中的对齐关系由 `alignment_gap` 进行刻画，该量在系统中作为诊断和导出字段保留，但默认不直接主导最终等级判定。

情感分支包含当前文本情感头和历史文本情感头。两者先在 record-level 输出 logits，再在区域级进行聚合。情感头实现随主模型一并位于 [multimodal.py](/home/chelizi/project/uniter/src/uniter/models/multimodal.py)。

## 五、空间指标 IFI

`IFI` 实现位于 [spatial.py](/home/chelizi/project/uniter/src/uniter/metrics/spatial.py)。当前版本的 `IFI` 由四类信息共同构成：街景形态代理、卫星形态代理、历史与规划文本驱动的目标约束，以及点位几何与多视角一致性约束。

街景侧主要表达道路宽度代理、围合强度、开放度、绿化覆盖和交通侵入等局部空间特征。卫星侧主要表达建成区轮廓、铺装廊道、绿化形态、边界连续性及俯视尺度下的结构关系。历史与规划文本则被解析为区域专属的目标空间要求，包括宽度、步行性、连通性、风貌展示、居民服务可达、消防通达、遗产风貌和绿化等目标项。点位几何部分使用 `point_id`、视角方向和经纬度恢复街巷观测顺序与几何连续性。

`IFI` 既输出单值指标，也输出组件级拆解结果。组件拆解可用于识别某一区域当前偏离主要来自街景侧、卫星侧还是规划约束侧。与 `IFI` 配套的 `spatial_proxy_score` 也定义在同一模块中，用于在缺乏大规模人工分割 mask 的情况下，对空间分支施加结构化训练约束。

从实现顺序上看，当前图像分支会先生成 segmentation logits，然后由 `compute_class_ratios()` 和 `compute_soft_class_ratios()` 之类的函数把像素级类别分布折叠成区域级语义比例。之后系统会把这些比例映射到课题需要的若干结构组，例如道路表面、围合、绿化、机动性相关要素和步行相关要素。街景组和卫星组会分别形成自己的 profile，再与默认 target profile 以及历史规划文本推导出的区域目标 profile 做比较。

历史与规划文本的处理不是简单把原文嵌入后直接和图像做余弦相似，而是先从文本中抽取与空间约束相关的关键词和结构条件，再通过 `build_historical_plan_targets()` 形成显式目标项。这样得到的目标项再与街景和卫星两侧的实际观测进行逐组比较，因此当前 `IFI` 的计算逻辑本质上更接近“基于文本约束的空间偏离分析”，而不是单纯的视觉特征差异度量。

点位几何部分的实现也是同样的思路。系统会根据 `point_id`、`view_direction` 和 `lon/lat` 信息恢复各街景点位在街巷中的相对位置关系，再构造街景几何连续性和跨视角一致性约束。这样，图像不再被当作完全无序的集合，而是被视为同一街巷空间的多位置、多方向观测样本。

## 六、身份指标 IAI

`IAI` 实现位于 [identity.py](/home/chelizi/project/uniter/src/uniter/metrics/identity.py)。系统首先从 `identity_texts` 中抽取结构化身份属性向量，再在此基础上生成身份 profile。当前 profile 由功能属性、界面属性和结构属性三个层面构成，并进一步计算混合度、冲突度、界面张力和结构张力等信号，最终汇总为区域级 `IAI`。

当前实现已经支持在导出层保留身份属性向量及其 profile，不仅输出单一 `IAI` 数值。对于单区域场景，这意味着身份维度既可以参与最终判定，也可以在结果解释中被展开分析。

从实现细节上看，身份文本并不是直接送入一个分类头后得到若干离散标签。系统会先在文本中识别功能角色、边界角色和结构角色，再把这些局部属性重新组织成区域级身份属性向量。随后通过 `_identity_confusion_score()` 一类内部函数，把身份属性之间的分布、冲突和张力转成可量化的混淆信号。因此，当前 `IAI` 的输入是文本，输出虽然是一个单值指标，但中间过程已经是结构化属性建模，而不是纯文本相似度计算。

## 七、意义指标 MDI

`MDI` 实现位于 [meaning.py](/home/chelizi/project/uniter/src/uniter/metrics/meaning.py)。当前默认模式为 `thesis`，即同时使用逐文本情感分布差与叙事语义漂移来构造区域级意义漂移指标。

逐文本当前情感标签和逐文本历史情感标签都会进入训练链路，并在区域级聚合后参与 `MDI` 计算。与此同时，系统还保留当前文本与历史文本在共享语义空间中的漂移信息，用于表达感知层与叙事层之间的偏差。最终 `MDI` 既包含监督信号驱动的情感差，也包含语义漂移信息，不依赖单一信息源。

在具体实现中，当前文本和历史文本先分别经过 record-level 情感头得到 logits，再通过区域索引进行聚合。系统不会只保留一个简单均值，而是会保留区域级情感分布信息，并在此基础上计算当前感知与历史叙事之间的分布差异。与此同时，编码后的当前文本区域向量和历史文本区域向量也会参与语义漂移计算。当前 `MDI` 的最终结构因此具有双通道特征：一条通道是监督情感差，另一条通道是无监督语义漂移。

## 八、训练机制

训练逻辑位于 [trainer.py](/home/chelizi/project/uniter/src/uniter/training/trainer.py)。当前训练目标由多个损失项构成，包括图像与文本对齐损失、身份损失、当前情感损失、历史情感损失、可选的 `lost_space` 分类损失、可选的 segmentation 损失，以及空间代理损失。

默认配置下，活跃的损失项主要是对齐损失、身份损失、当前与历史情感损失以及空间代理损失。`lost_space` 分类头默认关闭，正式分割监督也默认关闭，因此训练闭环主要依赖中间指标与空间代理监督，而不是依赖最终标签分类。

空间代理监督是当前训练闭环中的关键机制。由于现有数据并不包含大规模像素级分割标注，系统通过 `spatial_proxy_loss` 将历史与规划文本中解析出的目标结构，与图像分支提取到的空间代理形态进行比较，并将这种偏差作为训练约束。这样可以在没有正式 segmentation mask 的条件下，使空间分支仍然受到课题目标的显式约束。

训练器会自动识别当前 manifest 是否属于单区域 bootstrap。识别依据包括 `parent_region_id`、`bootstrap_view` 以及 split 后的区域记录结构。识别结果会同时影响日志提示、规则判定和阈值校准。

从训练流程看，区域级多模态 batch 在进入模型后会同时得到图像嵌入、文本嵌入、情感 logits、三个核心指标以及若干诊断字段。训练器随后一方面根据标签计算显式监督损失，另一方面根据中间指标计算规则相关的一致性量和空间代理约束，并把这些量统一纳入总损失。也就是说，当前训练并不是“先训练一个编码器，再离线算指标”，而是指标层本身已经和训练过程发生耦合，尤其体现在情感分支和空间代理分支上。

## 九、规则判定与阈值校准

规则判定模块位于 [judgement.py](/home/chelizi/project/uniter/src/uniter/inference/judgement.py)。当前默认使用 `thesis` 模式，最终等级由 `IFI / IAI / MDI` 三个指标通过规则映射得到。`alignment_gap` 默认不参与升级逻辑，`judgement_fusion` 默认关闭，因此系统默认输出的是规则结论，而不是黑盒融合结论。

单区域 bootstrap 场景下，规则系统采用保守化处理。当前逻辑将 `IFI` 和 `MDI` 视为更主要的锚定指标，将 `IAI` 视为支持性指标，从而避免在同一父区域拆分场景中因身份分支整体偏高而导致等级过度上抬。

阈值校准位于 [calibration.py](/home/chelizi/project/uniter/src/uniter/inference/calibration.py)。当前校准支持三种模式。样本数足够时按 quantile 估计阈值；样本数不足时采用 `bootstrap_shrinkage`；在单区域 bootstrap 且存在 `lost_space_label` 的情况下，再执行 `label_guided_search`，对 anchor 与 identity 阈值进行候选缩放搜索，使规则输出更贴近已知人工标签。

这种校准设计适合当前单区域小样本条件，能够避免阈值在极少样本下完全失稳。

## 十、评估与导出

评估器位于 [evaluator.py](/home/chelizi/project/uniter/src/uniter/inference/evaluator.py)。当前评估结果除了常规分类摘要外，还包含 `indicator_validation` 和 `bootstrap_stability` 两类结构化诊断。其中，`indicator_validation` 用于单独汇总 `IFI` 和 `MDI` 的代理验证结果，`bootstrap_stability` 用于检查同一父区域下多个 bootstrap 子区域在数值与最终等级上的一致性。

导出器位于 [exporter.py](/home/chelizi/project/uniter/src/uniter/inference/exporter.py)。当前导出产物包括区域级指标表、区域级结果表以及 Markdown 格式的结果摘要。导出的字段除 `final_level` 之外，还包括 `IFI / IAI / MDI`、`alignment_gap`、`spatial_proxy_score`、是否属于单区域 bootstrap、目标标签、身份属性向量和 IFI 组件级信息。

结果解释链路由 [context.py](/home/chelizi/project/uniter/src/uniter/reporting/context.py) 和 [region_results.py](/home/chelizi/project/uniter/src/uniter/reporting/region_results.py) 提供。当前解释层会根据 manifest 与图像路径自动回连图像索引、文本索引和元数据，并将这些上下文整合到结果导出与展示中。

当前输出层的组织方式是分层的。JSON 评估文件主要承担结构化诊断职责，CSV 结果表承担进一步分析和外部制图职责，Markdown 结果表则承担面向报告的快速汇总职责。实验目录根部还会额外生成 [artifacts_index.json](/home/chelizi/project/uniter/runs/train/kaitong-west-lane-single-region-rerun-optimized-20260406/artifacts_index.json)，用于集中索引 `checkpoints / summaries / evaluations / calibration / exports / visualizations`。由于输出字段中已经保留了区域标识、split、目标标签、三指标数值、身份属性向量和 IFI 组件信息，后续如果需要把结果继续接入 BI 工具、统计脚本或论文图表制作流程，当前导出格式已经具备可直接使用的基础。

## 十四、适用范围与边界

当前版本面向单区域研究与案例分析。它适合完成单区域下的多模态表征、指标计算、区域级判定和结果解释，也适合作为后续扩展到多区域研究前的技术基线。

当前版本不用于多区域统计推断，也不用于新增模态建模。声音、点云以及完整多区域统计基线不在本版本范围内。空间分支当前具备代理监督闭环，但不等同于大规模域内人工分割标注训练系统。单区域 bootstrap 结果适用于结构验证与方法分析，不直接外推为大样本泛化结论。
