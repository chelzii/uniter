# 单区域最终标签方案

- 人只需要判断并打标签
- 其他目录结构、字段结构、文件格式都提前定好
- 后续可以直接接入当前仓库

当前结论：

- **必打标签**：当前感知文本情感标签、历史文本情感标签
- **建议打标签**：区域级失落空间标签
- **可选增强**：少量图像分割 mask
- **不需要额外打标签**：身份文本

原因很简单：

- 单区域下，`lost_space_label` 作为主监督训练信号并不强，因为区域样本数太少
- 但文本样本很多，情感标签更适合先打
- `MDI` 本来就和“当前感知 vs 历史叙事”的情感差异强相关
- 身份文本现在已经作为输入存在，不需要再额外造一套身份监督标签

## 一、推荐标签优先级

### 1. 必打：当前感知文本情感标签

字段名：

- `sentiment_label`

标签空间：

- `0 = negative`
- `1 = neutral`
- `2 = positive`

适用对象：

- 当前感知文本
- 来源于微博、小红书、评论摘要等

为什么必打：

- 当前文本量最大
- 最适合做人判断
- 直接服务 `MDI`
- 比单区域里直接打 `lost_space_label` 更有训练价值

### 2. 必打：历史文本情感标签

字段名：

- `historical_sentiment_label`

标签空间：

- `0 = negative`
- `1 = neutral`
- `2 = positive`

适用对象：

- 历史文献摘录
- 规划文本摘录
- 碑志/文献中与当前区域相关的历史叙事

为什么必打：

- 当前区域历史文本条数少
- 全部人工打完成本低
- 能把 `MDI` 从“嵌入漂移代理”往“情感差异”推进

### 3. 建议打：区域级失落空间标签

字段名：

- `lost_space_label`

标签空间：

- `none`
- `light`
- `moderate`
- `severe`

建议同时保留数值映射：

- `0 = none`
- `1 = light`
- `2 = moderate`
- `3 = severe`

适用对象：

- 当前单区域 `kaitong_west_lane`

为什么是“建议打”而不是“必打”：

- 单区域下它更适合作为**最终人工判定/校验答案**
- 不太适合作为当前阶段的主监督训练目标
- 但如果后面你要做结果对照或论文表达，它仍然应该有

### 4. 可选增强：少量图像分割 mask

字段名：

- `segmentation_mask_paths`

适用对象：

- 一小批代表性街景图像

说明：

- 这不是纯“判断型标签”，而是图像标注
- 如果你现在只想让人做判断和选择，可以先不做
- 如果后面你愿意补图像标注，它能提升空间分支可信度

### 5. 不需要额外打：身份标签

当前 `identity_texts` 已经是输入数据。  
单区域下不建议再额外造一套身份监督标签。

当前更合理的做法是：

- 保证 `identity_texts` 内容准确
- 让它进入训练与 `IAI`
- 不再新增人工身份分类标签

## 二、怎么打最合适

### 1. 当前感知文本怎么打

判断标准：

- `negative`
  明确表达负面空间体验、秩序受损、噪声干扰、拥挤、破坏感、体验差
- `neutral`
  主要是事实陈述、时间信息、位置问答、非明显情绪表达
- `positive`
  明确表达安静、庄重、清净、好逛、氛围好、历史感强、值得去

不进入打标的数据：

- 明显说的是别的地方
- 明显与场所体验无关
- 纯互动寒暄
- 纯表情或无实义文本

### 2. 历史文本怎么打

判断标准：

- `negative`
  明确强调衰败、破坏、失序、功能退化
- `neutral`
  主要是事实性叙述、空间位置、制度说明、工程措施
- `positive`
  明确强调礼仪性、秩序感、历史价值、传统风貌、文化意义

注意：

- 历史文本不要按现代社交语气理解
- 它的“positive”更多是“积极历史价值/秩序性”
- 不是“开心/愉快”这种口语情绪

### 3. 失落空间标签怎么打

判断标准建议：

- `none`
  当前空间状态与历史意义、身份、空间形态基本一致
- `light`
  有轻度偏离，但整体仍可辨识其场所意义
- `moderate`
  偏离已较明显，至少两项指标上出现持续性问题
- `severe`
  空间形态、身份表达或意义感知发生明显断裂，场所特征已严重削弱

单区域建议：

- 这一个标签最好由你基于图像、文本、历史材料综合判断
- 也可以再加一列 `label_rationale`，简单写判断原因

## 四、每个文件怎么存

### 1. 当前感知文本标签文件

路径：

`datasets/annotations/kaitong_west_lane/current_sentiment/current_sentiment_labels.csv`

建议字段：

```csv
record_id,region_id,source_platform,source_type,normalized_date,text,sentiment_label,labeler,notes
```

字段说明：

- `record_id`
  对应原始清洗索引中的唯一 ID
- `region_id`
  当前固定为 `kaitong_west_lane`
- `source_platform`
  `weibo` / `xiaohongshu`
- `source_type`
  `comment` / `note`
- `normalized_date`
  标准化日期
- `text`
  原文本
- `sentiment_label`
  `0/1/2`
- `labeler`
  谁打的
- `notes`
  可空，记录歧义原因

### 2. 历史文本标签文件

路径：

`datasets/annotations/kaitong_west_lane/historical_sentiment/historical_sentiment_labels.csv`

建议字段：

```csv
record_id,region_id,source_file,page,text,historical_sentiment_label,labeler,notes
```

字段说明：

- `record_id`
  历史文本片段 ID
- `source_file`
  来源 PDF
- `page`
  页码
- `historical_sentiment_label`
  `0/1/2`

### 3. 区域级失落空间标签文件

路径：

`datasets/annotations/kaitong_west_lane/lost_space/region_lost_space_label.json`

建议格式：

```json
{
  "region_id": "kaitong_west_lane",
  "lost_space_label": "moderate",
  "lost_space_label_id": 2,
  "labeler": "your_name",
  "label_date": "2026-03-30",
  "label_rationale": "空间仍具历史街巷骨架，但慢行、停车、行政与居住叠合带来身份与意义偏移，已达到中度失落。"
}
```

### 4. 可选：分割 mask 标注目录

路径：

```text
datasets/annotations/kaitong_west_lane/segmentation/
├── image_index.csv
└── masks/
```

如果后面做，建议 `image_index.csv` 至少有：

```csv
image_id,image_path,mask_path,is_labeled,notes
```

## 五、从现有数据里举例

下面都用你当前真实数据里的例子，不是虚构的。

### 1. 当前感知文本打标例子

来源文件：

`datasets/data_workspace_cleaned/current_texts/current_text_index_cleaned.csv`

例子 A：

```text
卧龙禅寺是我去过最不商业的一个寺庙
```

推荐标签：

- `sentiment_label = 2`

原因：

- 明确正向评价场所气质
- 表达的是空间/场所体验

例子 B：

```text
下午五点关门
```

推荐标签：

- `sentiment_label = 1`

原因：

- 事实陈述
- 没有明确情感倾向

例子 C：

```text
人少清静
```

推荐标签：

- `sentiment_label = 2`

原因：

- 明确正向评价环境氛围

例子 D：

```text
洒金桥人车都一条街,体验感太差,差点进去出不来
```

处理建议：

- **如果确认评价对象不是 `kaitong_west_lane`，就不要进入打标文件**
- 这类样本应该在数据清洗时排除，而不是强行贴标签

### 2. 历史文本打标例子

来源文件：

`datasets/data_workspace_cleaned/historical_texts/kaitong_west_lane.json`

例子 A：

```text
同一规划在景观与交通层面将开通西巷的沥青路面改造为石板路面，并将其纳入街区内部步行街巷（与书院门步行街、卧龙寺巷等并列），以延续传统街巷风貌并服务慢行系统。
```

推荐标签：

- `historical_sentiment_label = 2`

原因：

- 明确强调传统风貌延续和慢行系统
- 带正向历史/规划价值导向

例子 B：

```text
规划中的管线综合按道路宽度分级：开通西巷与西号巷、县仓巷、安居巷支巷、大吉昌巷等同属约4—6米断面，宜布设给水、污水、电力、电信、燃气管线与雨水边沟。
```

推荐标签：

- `historical_sentiment_label = 1`

原因：

- 更偏事实性规划说明
- 不明显正面也不明显负面

### 3. 区域级失落空间标签例子

来源文件：

- 图像：`datasets/data_workspace_cleaned/images/kaitong_west_lane/`
- 当前文本：`datasets/data_workspace_cleaned/current_texts/kaitong_west_lane/kaitong_west_lane.txt`
- 历史文本：`datasets/data_workspace_cleaned/historical_texts/kaitong_west_lane.json`
- 身份文本：`datasets/data_workspace_cleaned/identity_texts/kaitong_west_lane.txt`

示例写法：

```json
{
  "region_id": "kaitong_west_lane",
  "lost_space_label": "moderate",
  "lost_space_label_id": 2,
  "labeler": "your_name",
  "label_date": "2026-03-30",
  "label_rationale": "作为历史文化街区内部连通街巷，当前仍保有一定历史骨架与寺院关联，但慢行、停车、居住与服务功能叠合造成身份与意义的偏移，综合判断为中度失落。"
```
